# Progress記録スキル

`docs/progress.md`に開発進捗・設計議論・トラブルシューティングを記録する。

## 記録フォーマット

```markdown
---

## YYYY-MM-DD: タイトル

### 背景

[何について議論/作業したか]

### 懸念1: [懸念の要約]

**問題**: [具体的な問題]

**解決**: [採用した解決策]

### 実装した変更

- [変更点1]
- [変更点2]

### 関連ドキュメント

- [関連ファイルへのリンク]
```

## 記録ルール

1. **日付**: `YYYY-MM-DD`形式
2. **懸念と解決**: ユーザーの懸念・質問は必ず「懸念N」として記録し、解決策とセットで記述
3. **検討した代替案**: 複数案を検討した場合は列挙
4. **実装変更**: 具体的なファイル・関数名を含める
5. **既存内容の後に追記**: ファイル末尾に追加
